# Git Commit Rules

## 1. Message Format
- Use Conventional Commit tags: `feat:`, `fix:`, `chore:`, `refactor:`, `docs:`, `test:`, `perf:`, `build:`, `ci:`.
- The subject line must be <= 72 characters and lowercase.

## 2. Clean Commits Only
- Never add co-author trailers, Claude attribution tags, or signed-off-by lines. Keep git history clean and professional.
